 How To

  • Learn to use git
    Learn to use git, commit early and often.
    Build your portfolio. Present your work on your website. (You can use GitPages)
  • Understand the problem
    Do your own research. It's incedibly useful skill which can be applied to and can be applied to anything else.
    If the problem seems difficult, good! Draw it out, write it out, do whatever you need to do to solve it.
    If the problem you're solving is too easy, brainstorm and try adding additional functionality. Try combining it with other ideas to make your software more challanging.
  • Implementation
    Take the language you're most familiar with to start your project. If the problem is too easy, pick a language you've never used before. Don't be afraid to write awful code and don't worry about the best implementation. Get a working implementation first, then refactor and clean it up.
  • Test
    Write test cases for your program. Get used to thinking about how you can break your software. Find problems, bugs and fix them. Repeat over and over again and document it. Throw it together in your portfolio (don't forget to commit)
  • Re-Implement
    Now that you have something together with lots of documentation and experience, try different language and try to match it as closely as possible to your first implementation. Add features, try new techniques, new styles. Go crazy!
  • Don't stop learning
    Now that you've gotten this far, don't stop trying to challange yourself. Keep a portfolio/blog/diary and keep adding to it.
